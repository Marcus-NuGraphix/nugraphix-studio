# Accessibility Checklist

Last updated: 2026-02-18
Status: Draft

## Goal

Enforce baseline accessibility across public and admin surfaces.

## Checklist

- [ ] Form fields have labels and error associations.
- [ ] Dialogs/sheets have titles and proper focus trapping.
- [ ] Keyboard navigation works for all actionable controls.
- [ ] Focus-visible styles preserved.
- [ ] Color contrast meets WCAG AA.
- [ ] Motion supports reduced-motion preference.

## Validation Scope

- Shared UI primitives first.
- Dashboard shell and navigation.
- Auth and content-management forms.
